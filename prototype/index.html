    Live public alpha

    The API is live without authentication during public alpha. It is release-pinned and rate-limited, but has no availability SLA yet.

    Core endpoints

    • GET/v1/search?q=interface+statusRanked discovery
    • GET/v1/objects/{objectId}Object semantics and provenance
    • GET/v1/modules?q=BFDRights-cleared module catalog
    • GET/v1/modules/{module}/rawLicensed raw MIB download
    • GET/v1/sourcesPublication mode and rights scopes
    • GET/v1/enterprises/{number}IANA PEN lookup
    • GET/v1/sys-object-ids/{oid}Evidence-bounded identity
    • GET/v1/modules/{module}/dependenciesDependency graph states
    • POST/v1/resolve:batchOrder-preserving OID resolution
    • GET/v1/data-releaseActive immutable release

    Public alpha contract

    • HTTPS JSON under a versioned /v1 namespace.
    • Every response identifies its immutable data_release.
    • Batch order and duplicate inputs are preserved.
    • Invalid OIDs and valid-but-not-found OIDs remain distinct.
    • PEN ownership never silently becomes a device-model claim.
    • Errors use RFC 9457 application/problem+json.
    1,000 OIDs / batch 64 KiB request body 200 search characters 120 requests / minute

    Public alpha limits may tighten during abuse; there is no availability SLA.
